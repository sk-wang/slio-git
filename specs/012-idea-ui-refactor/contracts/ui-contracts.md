# UI Contracts: IDEA Git 视图重构

**Branch**: `012-idea-ui-refactor` | **Date**: 2026-04-04

## 1. 提交历史右键菜单结构

按 IDEA Git.Log.ContextMenu 分组排列：

```
┌────────────────────────────────┐
│ 历史重写                        │
│   修改提交消息 (Reword)         │
│   Fixup 到此提交               │
│   Squash 到此提交              │
│   丢弃提交 (Drop)              │
│   压缩选中提交 (多选时可见)     │
│   交互式变基                    │
├────────────────────────────────┤
│ 提交操作                        │
│   还原提交                      │
│   撤销提交到此处 (Uncommit)     │
├────────────────────────────────┤
│ 引用操作                        │
│   从此提交新建分支              │
│   从此提交创建标签              │
│   推送到此提交                  │
├────────────────────────────────┤
│ 重置                            │
│   重置当前分支到此处            │
├────────────────────────────────┤
│ 复制                            │
│   复制提交哈希                  │
│   导出补丁                      │
└────────────────────────────────┘
```

## 6. 全文预览渲染

```
当 diff 为空且文件非二进制：
┌──────┬────────────────────────────────┐
│ 行号 │ 内容（全部标记为绿色新增行）     │
├──────┼────────────────────────────────┤
│  1   │ + use std::path::Path;         │
│  2   │ + use crate::error::GitError;  │
│  3   │ +                              │
│ ...  │ + (带语法高亮)                  │
└──────┴────────────────────────────────┘

当文件为二进制：
┌──────────────────────────────────────┐
│ ⚠ 二进制文件，无法预览               │
│ 文件路径: assets/logo.png            │
└──────────────────────────────────────┘

当文件超过 1MB / 5000 行：
┌──────────────────────────────────────┐
│ ⚠ 文件过大，仅显示前 5000 行         │
│ (全文大小: 2.3MB, 12000 行)          │
├──────┬───────────────────────────────┤
│  1   │ + ...                         │
│ 5000 │ + ...                         │
└──────┴───────────────────────────────┘
```
